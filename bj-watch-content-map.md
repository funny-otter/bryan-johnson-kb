The current site has some curated wiki content, but it is not yet the same information architecture as the reference prototype.

Primary mismatch:

1. Tweet material exists in `/home/chad/wiki`, but the site only uses a small curated subset in `src/data/curated-home.mjs`; it does not expose a recent tweet/signal feed, tweet-derived protocol updates, or real tweet telemetry.

3. The reference nav is: overview, health, longevity, nutrition, sleep, concepts, metrics, changelog. Current `src/layouts/BaseLayout.astro` nav is grouped as feed/protocols/reference/signal and lacks top-level `nutrition`, `sleep`, and `concepts` routes.

## Why tweets were not being used

There are two separate issues.

First, the wiki cron originally behaved like a digest monitor instead of an ingest workflow. This is documented in `/home/chad/wiki/log.md`:

- `2026-05-25 query | Bryan Johnson X post ingest status`: older 2026-05-22 X captures existed, but 2026-05-24 and 2026-05-25 daily digest post IDs were not present.
- `2026-05-25 ingest | Bryan Johnson X/Twitter daily batch backfill`: corrected the scheduled job intent and created `/home/chad/wiki/raw/articles/bryan-johnson/x-twitter-daily-2026-05-25.md`.

Second, the site does not have a tweet/signal data model. It has curated cards in `src/data/curated-home.mjs`, including some tweet-derived Enhanced Games, Kate Tolo, microplastics, certified partner, and Don’t Die positioning items. But tweets are not first-class content in the site; they are compressed into six overview cards. That means the prototype’s “Latest signal” rows, engagement columns, posting telemetry, topic mix, and changelog/protocol-update rows cannot be backed by real source data yet.

Recommendation: add a small curated signal data file, for example `src/data/signals.ts`, generated or manually curated from the raw wiki tweet captures. Do not dump raw tweets. Each signal should have:

```ts
{
  id,
  date,

  kind: 'tweet' | 'protocol_update' | 'product' | 'third_party',
  topic,
  title,
  summary,
  href,
  sources: [{ wikiPath, url, tweetId?, engagement? }],
  confidence: 'high' | 'medium' | 'low',
  medicalCaution?: string,
}
```

Use this file for overview feed rows, right-sidebar topic mix, recent tweets/signals, source counts, and changelog/protocol update extraction.
